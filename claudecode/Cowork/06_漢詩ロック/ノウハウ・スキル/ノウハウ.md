# AI動画編集の教科書 ── ノウハウまとめ

## 概要

**コンセプト**：「AIに日本語で指示するだけで動画を作る」
- 使うツール：コーディング支援AI（Antigravity / Claude Code）× Remotion
- 専門的なソフト操作やプログラミング不要。言葉 → AI（コード生成）→ Remotion（映像化）という流れ。

---

## 1. システム構成

| 役割 | ツール |
|------|--------|
| 指示を出す人 | あなた（日本語） |
| コードに翻訳するAI | Antigravity（無料・推奨）／Claude Code（月$20） |
| 映像を生成するツール | Remotion（React ベース。個人・3人以下は無料） |

### 推奨AIツール
- **Antigravity**：Google製、無料（使用量制限あり）。初心者向け。VS Codeベース。
- **Claude Code**：Anthropic製、Claude Pro（月$20～）が必要。高精度。

## 3. 基本操作の流れ

### プロジェクト作成
1. `~/AIVideoEditing/プロジェクト名` フォルダを作成（半角英数字推奨）
2. AIに「Remotionを使ってテキストアニメーションを作ってください」と依頼
3. AIが自動でRemotionセットアップ＋コード生成

### Remotion Studio の主要エリア

| エリア | 役割 |
|--------|------|
| 左パネル | Compositions（コンポジション一覧）・Assets |
| 中央 | リアルタイムプレビュー（ホットリロード対応） |
| 右パネル | Props（パラメータ調整）・Renders（書き出し状況） |
| 下部 | タイムライン・再生コントロール |

### Props の活用
- AIに「テキスト・色・サイズをPropsで調整できるようにして」と依頼
- Remotion Studioの右パネルでリアルタイムに値を変更可能
- フロッピーディスクアイコンで変更をコードに保存
- **役割分担**：AIが「仕組みを作る」→ 人間がPropsで「見た目を微調整」

### レンダリング
- Remotion Studio の「🚀 Render」ボタン または AIに「レンダリングしてください」と依頼
- 出力：プロジェクト内 `out/` フォルダに MP4 として保存

## 4. Remotion Skills（AIの専門知識強化）

### なぜ必要か
- AIはCSSアニメーション（`animate-bounce`など）を使おうとするが、RemotionはCSSアニメーション非対応
- Remotionはフレームベースで制御する必要がある
- Skillを入れることでAIがRemotionの作法を守ったコードを書く

### セットアップ
```bash
# Claude Code
npx skills add remotion-dev/skills

# Antigravity（チャットで依頼）
# 「Remotion公式のAgent Skills を .agent/skills/remotion/ に完全移植してください」
```

### 使い方
プロンプトに明示的に追加する：
> 「Remotion Skillsを参照して、〇〇を作ってください」
> 「.agent/skills/remotion/ のルールに従って実装してください」

### スキルのカスタマイズ
AIと解決した問題をスキルとして保存する習慣をつける：
> 「今回解決した『テキストの見切れ防止ルール』をスキルとして保存してください」

## 5. AIとの対話テクニック（4原則）

### ① 具体的に指示する（What / Where / When / How）

| 悪い例 | 良い例 |
|--------|--------|
| 「テキストをかっこよく動かして」 | 「0.5秒かけてフェードイン、1秒静止後、0.3秒で上にスライドアウト」 |
| 「ロゴを目立たせて」 | 「白い輪郭線をつけ、1秒間に1回のペースでゆっくり点滅させて」 |

**プロンプトフレームワーク**：
- What：何を（テキスト、ロゴ、背景）
- Where：どの位置、どこからどこへ
- When：いつ、何秒かけて
- How：どんな変化（透明度・スケール・回転）

### ② 段階的に育てる
1. Step 1：骨格（静的配置）
2. Step 2：動きを追加
3. Step 3：Propsで調整可能に
4. Step 4：イージングで洗練

### ③ AIと人間の役割分担

| AIが得意 | 人間が得意 |
|----------|-----------|
| コード生成・複雑な計算 | コンセプト決定・審美的判断 |
| 繰り返し作業の自動化 | 最終的な感性ベースの微調整（Props） |

### ④ 失敗を恐れない（Git活用）
- 実験前にコミット → 試す → 失敗したら巻き戻す
- AIに「〇〇という状態に戻してください」と指示するだけでOK

## 7. テキストアニメーションの種類

| 種類 | 概要 |
|------|------|
| バウンスアップ | 下から一文字ずつ飛び出す |
| 回転フェードイン | 180度回転しながらフェードイン |
| 波打つレインボー | sin波で上下＋HSLで色変化 |
| タイプライター | 一文字ずつ表示＋点滅カーソル |
| 爆発エフェクト | 中央から放射状に飛び散る |

### フォントの変更（Google Fonts）
```
「Google Fontsから『Potta One』を読み込んで適用してください」
「Propsにフォントファミリーの選択ドロップダウンを追加してください」
```
- ライセンスはGoogle Fontsの各フォントページで確認すること

### テキスト装飾
- **縁取り**：`「白い縁取りを2pxで追加して」`
- **影**：`「右下に少しずれた濃いグレーの影を」`
- **グラデーション**：`「上から下へ青から紺色に変化するグラデーションに」`
- **マスキング**：動画や画像を文字の形で切り抜く

```
# 動画素材
「background-movie.mp4を背景として画面いっぱいに、ループ再生で表示してください」
```

## 9. モーショングラフィックス

### 基本の4要素
1. 位置（Position）
2. 大きさ（Scale）
3. 回転（Rotation）
4. 透明度（Opacity）

### イージング（緩急）の指定方法
- 「ブレーキをかけるみたいにゆっくり止まる」（ease-out）
- 「最初はゆっくり、だんだん加速する」（ease-in）
- 「ボールが弾むように」（spring/bounce）

## 11. ゆっくりキャラクター（表情アニメーション）

### 構成パーツ（public/ に配置）
- 顔のベース、目（開き・半開き・閉じ）、口（閉じ・半開き・開き）

### アニメーション実装
```
口パク：「4フレームごとに開閉する、音声がない時は閉じたまま」
瞬き：「5秒に1回ランダムで瞬き」
呼吸：「3秒周期でSin波で上下3px揺れる」
統合：「口パク・瞬き・呼吸を統合したコンポーネントを作成し、位置・サイズをPropsで変更できるように」
```

## 16. Editor Kit（直感的な編集ツール）

### 機能
- 画像をドラッグ&ドロップで差し替え
- キーボードショートカットでテロップ位置・スケール調整
- 変更が自動でコードに保存（Save Server経由）

### 主要ショートカット
| キー | 機能 |
|------|------|
| 1 | Durationモード（シーン長さ調整） |
| 2 | TextYモード（テロップ位置） |
| 3 | Imageモード（画像位置・スケール） |
| N / P | 次/前のシーンへ |
| U | Undo |
| ] / [ | シーンを長く/短く |

## 重要な注意点

1. **CSSアニメーション非対応**：Remotion SkillsをAIに必ず導入する
2. **フレームベースで考える**：30fps動画なら30フレーム = 1秒
3. **ライセンス確認**：フォント・VOICEVOXキャラクター・Remotionの商用利用
4. **段階的に作る**：一度の指示で完璧を求めず、対話を重ねながら育てる
5. **Gitでセーブ**：実験前は必ずコミット
