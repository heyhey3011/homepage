## 3. 既知の残課題（公開後に順次対応）

- **吟猫ピッチマップの使い方ガイド（`apps/pitchmap/guide.html`）**: スクリーンショット画像が未挿入です。各ステップの「ここに画面写真が入ります」という枠（`.screenshot-slot`）に、実際の操作画面のスクリーンショットを差し込む必要があります。HTML内にコメントで差し替え例（`<img src="./guide-images/step2-recorder.png" ...>` など）を残してあるので、`guide-images/` フォルダを作って画像を置き、コメントを外すだけで反映できます。
- **漢詩辞典（`apps/kanshi-jiten/`）にポータルへ戻る導線がない**: 他の4ツール（教室マップ・吟士アーカイブス・詩吟大辞典・ピッチマップ）には「← heyhey 詩吟ポータル」という戻りリンクがありますが、漢詩辞典にはこのリンクが未実装です。ブラウザの「戻る」で対応はできますが、他ツールとの一貫性のため、後日追加をおすすめします。
- **旧ページの和紙テクスチャ画像が欠損**: `about.html` / `blog.html` / `contact.html` / `recommended-videos.html` / `how-to-record.html` の背景CSSが `必要画像/washi-texture.png` を参照していますが、このファイルは `必要画像/` フォルダに存在しません。背景装飾のみのため表示は崩れませんが（画像なしでも背景色で表示されます）、正式な画像を用意して `必要画像/washi-texture.png` として配置すると見た目が改善します。
- **`security-recommendations.md` に過去のAPIキーが平文で残っている**: リポジトリ直下の `security-recommendations.md`（80行目付近）に、以前使われていたYouTube APIキーの文字列がそのまま記載されています。これは今回の変更ファイルではなく、以前のコミットからそのまま残っているものです。**Cloudflare Pagesへ公開する前に、Google Cloud Consoleで該当キーを無効化（削除）した上で、このファイルの記載も削除するか、キー部分を伏字にすることを強くおすすめします。**

## 4. sitemap.xml について

`apps/pitchmap/guide.html`（使い方ガイド）は、今回 `sitemap.xml` には追加していません。
理由: guide.html は検索エンジンから直接見つけてほしいページではなく、「吟猫ピッチマップ」アプリ内から遷移する補助的なヘルプページという位置づけのためです。他のアプリ（詩吟大辞典・漢詩辞典など）についても、アプリ内部の個別ページはsitemapに含めておらず、各アプリのトップページ（`apps/pitchmap/` など）のみを掲載する方針で統一しています。もし検索流入を狙いたい場合は、後日 `sitemap.xml` に個別追加することも可能です。

## 5. Cloudflare Pages 公開手順（簡単メモ）

1. Cloudflareダッシュボード →「Workers & Pages」→「Pages」→「アプリケーションを作成する」。
2. 公開方法はどちらでもOK:
   - **GitHub連携**: GitHubリポジトリと接続し、ブランチ（`renewal` または `main`）を選ぶと自動デプロイされる。
   - **Direct Upload**: リポジトリのファイル一式をZIP等でそのままアップロードする方法。GitHub連携なしでも公開できる。
3. ビルド設定は不要です（静的HTMLサイトのため、ビルドコマンドは空欄のままでOK）。
4. **出力ディレクトリ（Build output directory）はリポジトリのルート（`/`）を指定**してください。`homepage/` のような1階層下のフォルダ名は指定しないでください（本チェックはこの `homepage` フォルダ自体をサイトのルートとして扱っています）。
5. 独自ドメインが決まったら、`assets/js/links.js` 冒頭のコメント（5〜10行目）にある通り、各HTMLの `<link rel="canonical">` と `og:url` に記載されているGitHub PagesのURL（`https://heyhey3011.github.io/homepage/`）を、新しいドメインに一括置換してください。
