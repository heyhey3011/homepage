# ツールのデータ・ソース対応表(TOOL-SOURCES)

このファイルは、詩吟ポータルサイト(homepage)に載っている各ツールについて、
「どこで作業・更新するのが正しいか」をまとめたものです。

以前は、同じツールのデータが `Cowork` フォルダの中に何か所にも散らばっていて、
「どっちが最新かわからない」状態になっていました。これを整理するためのメモです。

## 基本の考え方(いちばん大事なこと)

**これから先の更新は、すべて `homepage\apps` の中で直接おこないます。**

`凍結フォルダ\07_詩吟教室マップ【凍結】`、`凍結フォルダ\16_吟猫ピッチマップ【凍結】`、
`凍結フォルダ\02_詩吟の実演・動画アーカイブス【凍結】`、`凍結フォルダ\03_詩吟大辞典【凍結】` は、
今後は「凍結ラボ」として扱います(Cowork直下の `凍結フォルダ\` にひとつにまとめました)。つまり、

- 過去の試作・実験・素材を置いておく場所
- 新しいアイデアを試すときはここで自由に触ってOK
- **ただし、ここを直しても、サイトには反映されません**(デプロイ対象外)

サイトに反映したい変更は、必ず `homepage\apps\(ツール名)` の中のファイルを
直接編集してください。

例外は「かんし辞典(kanshi-jiten)」だけです。これは下の表と手順を参照してください。

## かんし辞典(kanshi-jiten)の更新手順

かんし辞典だけは特別で、「データカード」という元データのメモ(Markdownファイル)から
辞典のデータを組み立てる仕組みになっています。新しい漢詩を追加したいときは、
次の順番で作業してください。

1. **データカードを追加・修正する**
   場所: `00_ALL-SHIGIN\漢詩の図書館(データ工房)\v1\codex\DATA-CARD`
   ここに1つの漢詩につき1つのMarkdownファイル(データカード)を置きます。

2. **`parse_cards.py` を実行する**
   データカードを読み込んで、扱いやすい形式に変換します。

3. **`build_poems`(のビルド処理)を実行する**
   変換したデータから、辞典アプリが読み込む本番用データを組み立てます。

4. **`npm run build` を実行する**
   場所: `00_ALL-SHIGIN\漢詩の図書館(データ工房)\v1\claudecode\prototype-v2`
   Viteアプリをビルドして、`dist` フォルダに本番用ファイル一式ができます。

5. **`dist` フォルダの中身を `homepage\apps\kanshi-jiten` へコピーする**
   これでサイトに反映される状態になります。

   ※ この手順1〜5はまだ自動化されていません。手作業で行ってください。
   ※ `sync-tools.py` は、かんし辞典については「ビルドが必要なので手動でお願いします」という
   注意メッセージを出すだけで、コピーはしません(自動コピーすると壊れる可能性があるため)。

## サイトへの反映(デプロイ)手順

1. `homepage\apps\(ツール名)` の中のファイルを編集する
2. 変更を `main` ブランチにコミット・プッシュする
3. GitHub Actions が自動的に動いて、Cloudflare Pages に公開される
   (ワークフローファイル: `.github\workflows\deploy-cloudflare.yml`)

以前は GitHub Pages にも同時に自動デプロイされる設定になっていましたが、
二重デプロイでややこしいため停止しました。
`.github\workflows\deploy.yml`(GitHub Pages用)は、ファイルは残していますが、
自動では動かず、手動で実行したいときだけ動く設定に変更しています。

## 「凍結ラボ」フォルダとは何をする場所か

Cowork直下の `凍結フォルダ\` の中にまとめてあります。

- `凍結フォルダ\07_詩吟教室マップ【凍結】`
- `凍結フォルダ\16_吟猫ピッチマップ【凍結】`
- `凍結フォルダ\02_詩吟の実演・動画アーカイブス【凍結】`
- `凍結フォルダ\03_詩吟大辞典【凍結】`

(kanshi-jiten用のデータ工房 `00_ALL-SHIGIN\漢詩の図書館(データ工房)` は「凍結ラボ」ではなく、
現役で使うデータ元フォルダなので `00_ALL-SHIGIN` 直下に置かれています。)

これらは、今までの試作・実験・調査の記録として、そのまま残しています。
削除はしません(このプロジェクトのルール「ファイルは絶対に削除しない」の通りです)。

フォルダ名についている **【凍結】** は「もう更新しない・編集禁止」という目印です。
サイトに反映したい変更は、ここではなく必ず `homepage\apps\(ツール名)` 側を編集してください。

ただし、**ここを更新してもサイトには反映されません**。もし「うっかりこっちを更新してしまった」
という場合は、`sync-tools.py` を使って `homepage\apps` 側に取り込むことができます
(次のセクション参照)。

## 万が一のための同期スクリプト(sync-tools.py)

もし間違えて凍結ラボ側のファイルを更新してしまった場合の「救済用」スクリプトです。
普段の運用では使う必要はありません。

使い方は `sync-tools.py` のコメントを参照してください。
